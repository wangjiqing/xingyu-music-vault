# 星语音库 v1.3.3 - 歌词待办与覆盖率看板

## 发布摘要

v1.3.3 在 v1.3.0 / v1.3.1 / v1.3.2 的歌词草稿、逐字对齐、正式资产导入和歌词工作台体验基础上，新增歌词覆盖率看板、每日稳定歌词待办推荐、随机挑选待处理歌曲和歌曲列表统一歌词状态筛选。

本版本不新增歌词生成算法，不做网络歌词抓取，也不会自动批量创建歌词任务；所有入口都复用现有歌曲工作台、草稿确认和逐字对齐流程。

## 主要更新

- 首页新增“歌词资产进度”，展示可用歌曲总数、有可用歌词数、歌词覆盖率、已有 SWLRC 数、SWLRC 覆盖率、仅有 LRC、无歌词、制作中和待确认数量。
- 后端统一计算歌曲级歌词状态：`SWLRC_READY`、`LRC_READY`、`NO_LYRICS`、`ALIGNMENT_RUNNING`、`DRAFT_PENDING`、`FAILED`。
- 每天稳定推荐最多 5 首缺少 SWLRC 的歌曲，同一天刷新保持同一组结果。
- 推荐区分 `LRC_UPGRADE` 和 `NO_LYRICS`，不会把已有 LRC 的歌曲简单展示成“无歌词”。
- 推荐卡支持“开始制作”“今天跳过”“换一首”：开始制作跳转现有歌曲工作台，跳过仅当天隐藏，换一首会保留原推荐记录并持久化补位。
- 新增随机挑选待制作歌曲，支持 5 / 10 / 20 首，前后端均限制最大 20。
- 全部歌曲列表和歌曲卡片展示细分歌词状态 Badge。
- 全部歌曲列表支持按缺少 SWLRC、已有 SWLRC、仅有 LRC、无歌词、制作中、待确认和制作异常筛选。
- 首页“查看缺少 SWLRC 的歌曲”可带筛选条件跳转全部歌曲列表。

## 数据库迁移

新增迁移：

```text
V21__lyric_daily_recommendations.sql
V22__lyric_daily_recommendation_active_slot.sql
```

新增表：

```text
lyric_daily_recommendation
```

`V21` 新增每日推荐表，记录每日推荐日期、槽位、歌曲、推荐类型、操作状态、替换关系和操作时间，用于保证同一天推荐结果稳定。表内记录不代表永久忽略，也不会触发批量歌词制作。

`V22` 新增活跃槽位唯一索引。该索引必须独立于 `V21`，避免已经应用过 `V21` 的数据库出现 Flyway checksum mismatch。

新增索引包括：

- `idx_lyric_daily_recommendation_date`
- `idx_lyric_daily_recommendation_music`
- `idx_lyric_daily_recommendation_date_music`
- `idx_lyric_daily_recommendation_active_slot`

其中活跃槽位唯一索引用于避免并发请求生成同一天同槽位的重复推荐，同时保留 `REPLACED` 历史记录供审计。

## API 变更

新增管理端 API：

| Method | Path | 说明 |
|--------|------|------|
| GET | `/api/admin/lyrics/overview` | 歌词覆盖率与待办概览 |
| GET | `/api/admin/lyrics/recommendations/daily` | 获取当天可见稳定推荐 |
| POST | `/api/admin/lyrics/recommendations/{id}/start` | 标记开始制作 |
| POST | `/api/admin/lyrics/recommendations/{id}/skip` | 今天跳过 |
| POST | `/api/admin/lyrics/recommendations/{id}/replace` | 换一首 |
| POST | `/api/admin/lyrics/recommendations/random` | 随机挑选待制作歌曲 |

`GET /api/music` 新增歌曲级歌词状态字段：

```json
{
  "lyricStatus": "LRC_READY",
  "hasLrc": true,
  "hasSwlrc": false
}
```

`GET /api/music` 新增 `lyricStatus` 筛选参数，支持：

- `MISSING_SWLRC`
- `SWLRC_READY`
- `LRC_READY`
- `NO_LYRICS`
- `ALIGNMENT_RUNNING`
- `DRAFT_PENDING`
- `FAILED`

## 推荐规则

每日推荐候选必须同时满足：

- 本地音频文件存在且可用。
- 当前没有有效 SWLRC。
- 当前没有运行中的歌词对齐任务。
- 当前没有待确认歌词草稿。
- 同一天未被推荐过。

“换一首”会把原推荐标记为 `REPLACED`，并在原槽位创建新的 `PENDING` 推荐；“今天跳过”会把当前推荐标记为 `SKIPPED_TODAY`，当天不补位。

随机候选使用同一套缺少 SWLRC 候选口径，但不持久化、不创建任务，也不保证刷新后固定。

## 兼容性说明

- 保留旧歌词绑定详情 API 的 `BOUND`、`NO_LYRIC` 等状态语义。
- 管理端首页、歌曲列表、歌曲卡片和歌曲工作台使用 v1.3.3 歌曲级歌词状态。
- 旧 `hasLyrics` 查询参数仍保留兼容，但管理端前端优先使用 `lyricStatus`。
- OpenAPI 客户端同步契约不新增公开状态字段。
- 原有歌词草稿、对齐任务、审核导入、播放器和歌词工作台流程继续复用。

## 不包含内容

- AI 智能推荐。
- 自动批量制作歌词。
- 网络歌词抓取。
- 播放行为推荐。
- 永久忽略歌曲。
- 整库一键生成歌词。
- 新的歌词编辑器或第二套歌词工作台。

## 升级建议

1. 备份 `data`、`lyrics`、`artwork` 和现有数据库。
2. 更新镜像版本到 `v1.3.3`。
3. 启动服务后确认 Flyway 已应用 `V21__lyric_daily_recommendations.sql`。
4. 进入首页检查“歌词资产进度”和每日推荐区块。
5. 在全部歌曲列表验证 `lyricStatus=MISSING_SWLRC` 筛选结果。
6. 对已有 LRC 和完全无歌词歌曲分别进入歌曲工作台，确认仍走现有草稿 / 对齐工作流。
