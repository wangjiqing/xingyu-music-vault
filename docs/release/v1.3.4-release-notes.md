# v1.3.4 Release Notes

v1.3.4 聚焦歌词任务可观测性接入，推荐配合 `xingyu-lyrics-aligner:0.6.0` Worker 使用。

## 新增

- 后端兼容 Worker v0.5.0 `status.json`，读取阶段、心跳、attempt、配置、警告、错误和结果摘要。
- 新增 `GET /api/admin/lyric-alignment/jobs/{id}/observability`，统一返回草稿提取与逐字对齐任务的可观测详情。
- 读取 Worker 任务目录下最近 50 条 `events.jsonl` 事件；事件不入库，不做搜索。
- 任务列表和详情响应新增轻量 `observabilitySummary`。
- 草稿提取 Worker request 升级为 `schemaVersion: 3`，默认 preset 为 `RECOMMENDED`。
- 管理端歌词任务列表展示 Worker 阶段、心跳、运行耗时、preset、警告和错误摘要。
- 管理端任务详情新增 Worker 可观测性 tab，可查看状态、心跳、配置摘要、输出文件、最近事件和兼容性提示。
- 歌曲工作台的草稿任务和对齐任务展示轻量阶段 / 心跳 / 失败摘要。
- 草稿提取表单新增 FAST、RECOMMENDED、HIGH_QUALITY、FULL_RECOGNITION preset；高级参数默认折叠。
- 已完成的对齐任务新增「试听匹配」入口，可在工作台临时加载任务 LRC / SWLRC，随原曲检查滚动和逐字高亮，试听不会覆盖当前歌词。
- 消费 Worker v0.6.0 的 `firstAlignedLyricStartMs`、`preservedHeaderLines` 和 `presentationHints`，在导入前试听与导入后工作台中按前奏提示展示非演唱头部；旧任务缺少协议字段时自动降级。

## 兼容

- 保留旧 Worker `schemaVersion` 1 / 2、marker-only 任务和旧 request schema 兼容。
- `status.json` 损坏、未知 schema、任务目录缺失或 `events.jsonl` 缺失不会导致接口 500。

## 不包含

- 本版本不引入 WebSocket / SSE，不读取 stdout / stderr 内容，不做 events 入库、分页或搜索，不做任务重试和多 Worker 调度。
