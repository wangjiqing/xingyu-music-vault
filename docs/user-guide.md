# 用户手册

## 主题切换

管理后台 header 右侧提供轻量主题切换入口，当前覆盖四套四季主题：

- 春日晨光 / Spring Dawn
- 仲夏星河 / Midsummer Starlight
- 秋日唱片 / Autumn Vinyl
- 冬夜雪境 / Winter Moonlight

切换后，页面背景、Logo、favicon、主题变量 CSS 与页面空状态图会随当前主题更新。主题选择保存在浏览器 `localStorage` 中，刷新页面后会恢复上次选择；默认主题仍为仲夏星河 / Midsummer Starlight。

## 全部歌曲

全部歌曲是系统的核心页面，承载音乐库的日常浏览和整理工作。

### 切换视图

页面右上角有视图切换按钮，支持：

- **表格视图**：适合整理场景，完整展示文件名、歌手、专辑、年份、流派、歌词状态、封面状态
- **卡片视图**：适合浏览场景，以封面为核心展示歌曲，卡片包含歌名、歌手、专辑、时长

两种视图共享筛选条件，可随时切换。

### 卡片视图展示内容

每张卡片展示：

- 封面缩略图（无封面时显示默认占位图）
- 歌曲名称
- 歌手
- 专辑
- 时长

### 卡片快捷操作

卡片底部提供以下操作入口：

- **编辑**：打开元数据编辑弹窗，可修改 title、artist、album、year、trackNo、genre
- **歌词**：有歌词绑定的歌曲可点击查看歌词详情弹窗
- **更多操作**（...）：提供选择/导入封面、取消封面绑定、移入回收站

### 筛选与搜索

- **关键词搜索**：输入歌名、歌手或专辑名进行过滤
- **待整理 / 已整理**：按元数据完整度筛选
  - 待整理：元数据不完整的歌曲（缺少歌手、专辑等信息）
  - 已整理：元数据完整的歌曲

### 批量编辑

在表格视图中勾选多首歌曲，可批量填写共同字段（artist、album、year、genre）。title 和 trackNo 建议单首编辑。保存前会提示本次操作影响多少首歌曲。

### 左侧导航说明

当前左侧导航入口：

- **全部歌曲**：当前页面，表格/卡片视图
- **歌曲工作台**：只读校验页面，可边播放当前歌曲，边查看元数据、歌词、封面和 OpenAPI 输出预览
- **歌词**：进入歌词管理页面
- **封面**：进入封面管理
- **歌手**：进入歌手浏览页面（v0.8.1），点击歌手卡片进入歌手详情页（v0.8.2）
- **专辑**：进入专辑浏览页面（v0.8.3），可进一步进入专辑详情页
- **设置**：系统设置

以下入口已从导航中移除，不再显示在侧边栏：

- 待处理队列、扫描任务、音乐文件（路由仍保留，后台可访问）

全部歌曲页面右上角有**回收站**入口，可查看已删除音乐。

## 歌曲工作台（v1.2.1）

歌曲工作台用于本地 / 局域网可信环境下的人工校验：从左侧导航进入，或在「全部歌曲」列表点击「工作台」打开某首歌。页面左侧是当前工作台列表，右侧是当前歌曲工作区。

工作区包含：

- **播放器**：基于浏览器 HTML5 audio，支持播放 / 暂停、进度展示、当前时间 / 总时长、进度跳转、上一首 / 下一首。上一首 / 下一首仅在当前工作台列表内切换。
- **播放模式**：支持单曲循环、顺序列表、顺序循环、随机播放；非单曲循环模式下播放结束会按当前工作台列表自动切换下一首。
- **元数据**：展示数据库中的 ID、标题、歌手、专辑、专辑歌手、年份、流派、音轨号、时长、文件路径、歌词状态、封面状态和更新时间。
- **歌词**：有歌词时显示歌词正文；无歌词时显示「暂无歌词」空状态。
- **封面**：有封面时显示封面图片与基础信息；无封面时显示空状态；加载失败时显示错误。
- **OpenAPI 输出**：格式化展示当前歌曲面向客户端 OpenAPI 的预览 JSON，用于核对音乐盒等客户端看到的数据。

v1.2.1 工作台只读查看，不提供确认状态、AI 候选、网络刮削、歌词公网请求、封面写入、音频 Tag 写回、播放历史或收藏系统。

### 歌手浏览

歌手浏览页面（`/artists`）按歌手名聚合音乐库中的歌曲，提供以下能力：

**搜索与排序**：可按歌手名模糊搜索，支持按歌曲数降序、名称升序、专辑数降序、待整理数量降序排序。

**卡片展示**：每张歌手卡片展示歌手名称、歌曲数、专辑数、歌词数量、封面数量、待整理数量。

**分页浏览**：支持分页，每页条数可配置。

**跳转专辑详情**：点击歌手卡片进入歌手详情页，专辑分组中可点击专辑卡片跳转到对应专辑详情页（v0.8.3）。

### 专辑浏览（v0.8.3）

专辑浏览页面（`/albums`）按专辑名聚合音乐库中的歌曲，提供以下能力：

**搜索与排序**：可按专辑名或专辑歌手名模糊搜索，支持按歌曲数降序、名称升序、年份降序、待整理数量降序排序。

**卡片展示**：每张专辑卡片展示专辑名、专辑歌手、年份、歌曲数、歌词数量、封面数量、待整理数量。专辑封面由系统基于专辑名生成彩色首字母占位图（v0.8.3），后续版本将支持真实封面预览。

**分页浏览**：支持分页，每页条数可配置。

### 专辑详情（v0.8.3）

专辑详情页面（`/albums/detail?albumKey=...&artistKey=...`）展示单张专辑的详细信息：

**统计概览**：专辑名、专辑歌手、年份、歌曲数、歌词数、封面数、待整理数量。

**曲目列表**：通过 `GET /api/music?albumKey=...&artistKey=...` 获取该专辑包含的所有曲目，含歌曲名称、歌手、时长、歌词状态、封面状态。

可从歌手详情页的专辑分组点击跳转进入。

### OpenAPI 客户端接入（v0.9.0）

v0.9.0 为播放器客户端提供只读 OpenAPI，前缀为 `/api/open/v1`。本节说明客户端接入星语音库的建议流程和注意事项。

### 接入流程

**第一步：验证服务可用性。** 客户端启动时调用 `GET /api/open/v1/server/info`，确认 `serviceVersion` 和 `apiVersion` 与客户端兼容。

**第二步：检查音乐库是否变化。** 调用 `GET /api/open/v1/sync/state`，获取 `lastUpdatedAt`（活跃歌曲 track_files.updatedAt 与关联 tracks.updatedAt 的最大值）。若与本地缓存的值一致，可跳过全量数据拉取；若不一致，按需重新拉取曲目列表。

**第三步：拉取曲目列表。** 调用 `GET /api/open/v1/tracks`，使用分页参数（`page` + `pageSize`）逐页拉取，不要一次请求全量。列表项中 `artworkUrl` 为相对路径（如 `/api/open/v1/tracks/1/artwork`），客户端使用服务端 base URL 拼接，不是 base64。

**第四步：按需拉取歌词和封面。** 歌词和封面正文不随列表接口返回，需要时单独调用：
- `GET /api/open/v1/tracks/{id}/lyrics` — 歌词原文
- `GET /api/open/v1/tracks/{id}/artwork` — 封面文件 URL

**第五步：本地缓存。** 封面接口设置 `Cache-Control: max-age=3600` 和 ETag，客户端可利用条件请求（`If-None-Match`/`If-Modified-Since`）避免重复下载。歌词接口不设置 HTTP 缓存头，建议客户端直接本地缓存歌词内容。

**第六步：关联本地音乐。** 若客户端持有本地音乐文件，可通过 `GET /api/open/v1/match/track?title=...&artist=...` 查询服务端对应曲目 ID，建立关联后再使用其他接口。

### 按需拉取原则

- 歌词和封面正文不要在曲目列表接口中加载，也不要在列表项中嵌入 base64
- 歌词元数据（`/lyrics/meta`）用于判断是否有歌词、时间戳、语言，可用于快速过滤，无需下载正文
- 封面元数据（`/artwork/meta`）包含宽高、MIME、文件大小，可用于预留布局尺寸

### 分页拉取示例

```http
GET /api/open/v1/tracks?page=0&pageSize=50&keyword=周杰伦
```

翻页时递增 `page` 参数，直至 `total` 达到预期或 `items` 为空。

### 暂不支持的操作

以下能力在 v0.9.0 不提供，客户端不应尝试调用：

- 音频流播放
- 客户端修改元数据
- 上传音乐文件
- OAuth / JWT 认证（当前使用后端全局 Bearer Token，不需要 OAuth/JWT）
- WebSocket 推送
- 远程扫描音乐库

### 元数据同步（v0.8.4 / v0.8.5 / v0.8.6 / v0.8.7）

v0.8.4 新增音频文件内嵌 Tag 读取与双向同步能力，v0.8.5 补充审计历史与回滚基础能力，v0.8.6 补强参数校验、强化边界限制、明确回滚规则与风险说明，v0.8.7 完成 v0.8 功能冻结与回归测试，确认完整功能链路稳定可用。

**同步范围：** 当前仅同步歌曲名（title）、歌手（artist）、专辑（album）三个核心字段。专辑歌手、年份、流派、音轨号等高级字段暂不参与同步。

**支持格式：** mp3、flac、wav、m4a、aac、ogg、opus。

#### 操作流程

**第一步：差异比较。** 在歌曲列表中选中歌曲，点击「同步」操作。系统会比较该歌曲数据库中保存的元数据与音频文件内嵌 Tag 的差异，并展示对比结果。

**第二步：预览差异。** 差异比较会列出所有不一致的字段，例如：

- `artist`：数据库 = "周杰伦"，文件 Tag = "Jay Chou"
- `album`：数据库 = "叶惠美"，文件 Tag = "Ye Hui Mei"

用户确认后选择同步方向。当前差异对比范围仅限于歌曲名、歌手、专辑三个字段。

**第三步：执行同步。** 支持两种方向：

- **文件 Tag → 数据库**：将音频文件中的元数据覆盖到数据库记录。适用于从文件 Tag 补充数据库缺失信息。
- **数据库 → 文件 Tag**：将数据库中编辑好的元数据写回音频文件。适用于统一修正文件 Tag。

#### 批量操作限制

批量同步最多支持 100 首歌曲同时进行。批量回滚同样最多 100 条。超出限制时接口返回 `400 Bad Request`，拒绝执行。

#### 风险提示

**⚠️ 数据库写回音频文件会修改本地音频文件。执行前请确认文件已备份。v0.8.5 已提供审计历史页面与回滚能力。**

**写入格式限制：** `db_to_file`（数据库 → 文件 Tag）仅支持 jaudiotagger 3.0.1 已验证写入格式：mp3、flac、m4a、ogg（oga）、wav。aac 和 .opus 文件不支持写入，操作会返回明确失败，不会损坏文件。

**回滚说明：** 回滚是基于历史快照执行一次新的同步操作，不是删除历史。回滚同样会生成新的审计记录（`rollbackOfAuditId` 指向前序记录），形成操作链，可追溯。

**回滚规则：** 可回滚的审计记录需同时满足：`status = SUCCESS` 且 `rollbackStatus = NOT_ROLLED_BACK`。以下记录不可回滚：失败记录（`status = FAILED`）、已回滚记录（`rollbackStatus = ROLLED_BACK`）、ROLLBACK 记录（`rollbackOfAuditId != null`）。

**操作确认：** 所有覆盖操作和回滚操作都需要预览和用户确认，UI 展示明确的风险提示。

#### 后续版本可考虑

- 高级字段同步（albumArtist/year/genre/trackNumber）
- 封面写入音频文件
- 网络刮削元数据（MusicBrainz 等在线元数据源）
- AI 元数据补全

**v0.8.7 暂不支持：** 全库同步、全库回滚、按歌手一键同步、按歌手一键回滚。如需大规模同步，建议分批调用，每批不超过 100 条。
