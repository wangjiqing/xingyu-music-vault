# 星语音库 v1.3.0 - 歌词草稿与逐字对齐任务闭环

## 发布摘要

v1.3.0 新增一条面向管理端的人工辅助歌词生产链路：本地音频可以先通过独立 Worker 生成候选歌词草稿，管理员边听边校对后确认成可信歌词；可信歌词再作为逐字歌词对齐任务输入，Worker 输出 LRC / SWLRC，管理员审核后确认导入。

本版本的重点是“可追溯、可审核、可回滚”的管理流程，不是自动把模型输出直接发布给客户端。

## 主要更新

- 新增歌词草稿提取任务 `LYRIC_DRAFT_EXTRACTION`。
- 新增草稿校对工作台，支持边听边校对、保存草稿、恢复原始草稿、驳回和确认可信歌词。
- 新增 `DRAFT_CONFIRMED` 来源可信歌词资产，可作为逐字对齐任务输入。
- 新增歌词对齐任务创建、共享目录输入写入、READY 信号和 Worker 状态同步。
- 接入 `wangjiqing/xingyu-lyrics-aligner:0.4.0` Worker，支持草稿提取与逐字对齐两类任务。
- 新增 LRC / SWLRC / report / alignment 结果预览。
- 新增人工审核通过 / 驳回与确认导入流程。
- 导入后创建 `ALIGNMENT` 来源 LRC 记录，SWLRC 作为可选逐字歌词附加资产。
- OpenAPI 保持旧 LRC 客户端兼容，并为支持方提供可选逐字歌词信息。
- Docker Compose 示例支持音库 + Worker 双容器部署。

## 完整使用链路

候选歌词草稿：

```text
本地音频
→ Worker 生成未对齐候选文本
→ 管理端边听边人工校对
→ 管理员确认
→ 生成可信歌词资产
```

逐字歌词对齐：

```text
可信歌词 + 本地音频
→ Worker 生成 LRC / SWLRC
→ 管理端预览结果
→ 管理员人工审核
→ 确认导入
→ LRC 成为当前歌曲可用歌词，SWLRC 成为附加逐字歌词资产
```

## Docker 升级说明

推荐使用精确版本镜像：

```dotenv
IMAGE_TAG=v1.3.0
ALIGNMENT_WORKER_IMAGE=wangjiqing/xingyu-lyrics-aligner:0.4.0
```

首次启用 Worker 前需要准备共享任务目录和模型缓存目录：

```bash
mkdir -p alignment-jobs alignment-models
```

音库容器挂载：

```text
ALIGNMENT_JOBS_DIR:/alignment-jobs
MUSIC_DIR:/music:ro
```

Worker 容器挂载：

```text
ALIGNMENT_JOBS_DIR:/jobs
MUSIC_DIR:/music:ro
ALIGNMENT_MODELS_DIR:/models
```

Worker 启动命令：

```bash
xingyu-align worker run --jobs-dir /jobs --music-dir /music --device cpu
```

Worker 不暴露端口，不挂 Docker Socket，默认以 `10001:10001` 运行。

## Worker v0.4.0 依赖说明

v1.3.0 的歌词草稿提取依赖 `xingyu-lyrics-aligner:0.4.0` 的 v2 `LYRIC_DRAFT_EXTRACTION` 协议。逐字歌词对齐继续支持既有对齐协议，并由同一个 Worker 容器处理。

Worker 默认 CPU 模式。草稿提取和逐字对齐耗时取决于音频长度、模型缓存、机器性能和是否需要首次模型下载。首次运行可能需要模型预热或下载，请确保 `ALIGNMENT_MODELS_DIR` 有足够磁盘空间。

## 重要边界

- 草稿不是正式歌词，不会直接面向普通客户端输出。
- 确认可信歌词不会自动替换当前 LRC / SWLRC。
- 确认可信歌词不会自动创建逐字对齐任务。
- 对齐结果必须人工审核通过后才能确认导入。
- 导入不会删除或覆盖原始可信歌词。
- 导入后的正式歌词不会长期引用 Worker jobs 中间目录。
- 本版本不包含在线逐字时间轴编辑器。
- 本版本不包含批量全库转写、批量审核或批量导入。
- 本版本不包含自动确认可信歌词或自动创建对齐任务。
- 本版本不包含 GPU / 多 Worker 调度、Docker Socket、HTTP 回调、消息队列。
- 音库容器不安装 Python、WhisperX 或 PyTorch。

## 已知限制

- ASR 草稿质量取决于音频、人声分离、模型和歌曲语言，仍需要人工校对。
- Worker 默认 CPU 模式可能较慢，首次运行可能触发模型下载。
- 当前 `ALIGNMENT` LRC / SWLRC 已从 Worker jobs 目录复制到 `alignment-assets-dir`；后续版本会继续收口到 `LYRICS_DIR` / `MUSIC_VAULT_LYRIC_DIRS` 下的受控 alignment 子目录。
- 当前不提供在线逐字时间轴拖拽编辑。
- 当前不提供批量草稿提取、批量审核或批量导入。

## 升级建议

1. 备份 `data`、`lyrics`、`artwork` 和现有数据库。
2. 更新 `IMAGE_TAG=v1.3.0`。
3. 确认 `ALIGNMENT_WORKER_IMAGE=wangjiqing/xingyu-lyrics-aligner:0.4.0`。
4. 准备 `ALIGNMENT_JOBS_DIR` 和 `ALIGNMENT_MODELS_DIR`。
5. 启动后检查 `/api/health` 返回 `version=1.3.0`。
6. 首次使用草稿提取或逐字对齐时，观察 Worker 日志确认模型缓存和任务领取正常。

## 后续方向

- 正式对齐歌词资产目录迁移到歌词扫描目录下的受控 alignment 子目录。
- 扫描器识别 `ALIGNMENT` 资产，避免重复创建 `LOCAL_FILE`。
- 删除同步区分外部扫描歌词与音库生成歌词。
- 持续优化草稿提取质量、诊断信息和任务耗时提示。
