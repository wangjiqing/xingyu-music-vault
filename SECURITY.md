# Security Policy

Xingyu Music Vault is still in the v0.x stage. Security handling will continue to mature before v1.0.

## Reporting a Vulnerability

Please report security concerns through GitHub Issues, or contact the maintainer privately through the channels listed on the project profile if the issue should not be public.

Do not include sensitive information in public issues, including:

- real tokens or passwords
- real `.env` files
- SQLite database files
- personal music library paths
- private hostnames, IP addresses, or deployment details

If you need to describe a sensitive value, replace it with a placeholder such as `change-me`, `/path/to/music`, or `example.local`.

## Supported Versions

During the v0.x phase, security fixes are generally made on the latest development line. Older tags may not receive separate backports.
