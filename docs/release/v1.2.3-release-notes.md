# v1.2.3 Release Notes — 歌词扫描删除同步修复

v1.2.3 修复现有「扫描歌词」入口在本地 LRC 源文件删除后的同步问题。此前 `.lrc` 文件从 lyricsDir 中移除后，重新扫描歌词只会处理仍存在的文件，不会清理数据库中旧的 `lyrics` / `song_lyrics` 绑定，导致 OpenAPI 仍可返回旧歌词正文。

## 问题背景

星语音库的歌词扫描会把本地 LRC 内容导入数据库，并通过 `song_lyrics` 建立歌曲主歌词绑定。这个设计让客户端可稳定读取歌词正文，但也要求扫描逻辑在本地歌词目录变化后同步数据库状态。

v1.2.2 及更早版本中，新增或更新 LRC 可以正常导入和绑定；删除 lyricsDir 中已有 LRC 后，再次点击管理端「扫描歌词」不会解除旧绑定。因此音乐盒或其他 OpenAPI 客户端仍可能通过 `/api/open/v1/tracks/{id}/lyrics` 读取到已经从本地目录删除的旧歌词内容。

## 修复行为

- 保留现有管理端「扫描歌词」入口，不新增页面入口、扫描 Job 类型或音乐盒改动。
- lyricsDir 作为当前本地歌词可用性的事实来源。
- 扫描开始时收集当前扫描目录中实际存在的 `.lrc` 文件。
- 扫描完整成功且 `failed=0` 时，检查当前扫描 lyricsDir 范围内的 `LOCAL_FILE source_path`。
- 若数据库歌词记录的源文件已不存在，则解除对应 `song_lyrics` 绑定。
- 旧 `lyrics` 记录可作为无绑定历史记录保留；OpenAPI 可用性由当前主绑定决定。
- 删除同步后，`/api/open/v1/tracks/{id}/lyrics/meta` 返回 `available=false`，`/api/open/v1/tracks/{id}/lyrics` 返回 `404`。
- 若同内容或新内容 LRC 重新放回 lyricsDir，再次扫描会重新导入或复用去重记录并恢复主绑定。

## 安全边界

- 只有本次歌词目录遍历完整、文件处理无失败时，才执行缺失源文件解绑。
- 扫描目录不可访问、遍历中断或存在文件读取/解析失败时，不会把数据库中的既有歌词视为已删除。
- 删除同步只处理 `source_path` 位于当前扫描 lyricsDir 范围内的本地歌词记录。
- 路径判断会做规范化处理，并兼容已存在文件的真实路径解析。

## 验证方式

后端回归测试覆盖以下路径：

```bash
cd backend
mvn -q -Dtest=LyricResourceTest,OpenApiResourceTest test
```

重点场景：

- 已导入歌词后删除对应 LRC，再执行现有 `/api/lyrics/scan`。
- 验证被删除源文件对应歌曲的 OpenAPI meta 返回 `available=false`。
- 验证正文接口不再返回旧 `lyrics.content`。
- 验证其他歌曲仍在使用的歌词绑定不受影响。
- 放回新内容 LRC 后重新扫描，OpenAPI 恢复可用。
- 放回完全相同内容 LRC 后重新扫描，可通过去重记录重新绑定。
- 扫描中存在失败文件时，跳过删除同步以避免误解绑。

## 兼容性说明

- 不包含数据库迁移。
- 不改变 OpenAPI 路径、认证方式或响应结构。
- 不引入 IMPORTED / LINKED 双模式。
- 不修改星语音乐盒客户端逻辑。
- 管理端仍通过既有「扫描歌词」按钮触发同步。
- 客户端已缓存的歌词仍遵循客户端缓存策略；服务端增量变更会记录 lyrics 变更，客户端按既有同步流程刷新即可。
