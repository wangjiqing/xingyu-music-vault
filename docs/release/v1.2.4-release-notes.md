# v1.2.4 Release Notes

**发布日期：** 2026.06.21

v1.2.4 修复 GitHub Issue #51：歌词源文件删除后扫描会解绑旧记录，但同一路径 LRC 恢复或更新后再次扫描会新建歌词记录，导致旧未绑定记录永久遗留。

## 变更摘要

- 歌词扫描优先按规范化后的 `source_path` 识别同一源文件。
- 同一路径 LRC 恢复或内容变化后，复用原 `lyrics` 行并更新正文、内容哈希、标题、歌手、解析状态和更新时间。
- 恢复后重新执行歌曲匹配，恢复 `song_lyrics` 主绑定。
- 内容哈希去重保留为无同路径记录时的兜底逻辑。
- 歌词管理列表仅对未绑定记录显示「删除记录」操作。
- 新增 `DELETE /api/lyrics/{id}`，后端删除前再次确认该歌词未被 `song_lyrics` 引用。
- 删除歌词记录只删除数据库记录，不删除 Lyrics 目录中的 `.lrc` 源文件。

## 数据兼容性

本版本不新增数据库迁移，不做高风险全库清理。

对于历史重复数据，扫描仅在遇到同一规范化 `source_path` 且能确认重复记录未绑定时清理重复行；无法安全判断的记录会继续保留，管理员可在歌词管理页手动删除未绑定记录。

## 版本接口

- `/api/health` 返回版本更新为 `1.2.4`。
- `/api/admin/server/info` 的 `serviceVersion` 更新为 `1.2.4`。
- `/api/open/v1/server/info` 的 `serviceVersion` 更新为 `1.2.4`。
- 后端 Maven 构建版本、OpenAPI 文档版本、前端包版本和 image compose 默认 tag 同步为 `1.2.4`。

## 验证重点

- 已绑定 LRC 删除后扫描会解除绑定。
- 同一 `source_path` LRC 恢复后复用原歌词记录，不再新增重复行。
- 恢复后的内容变化会更新原记录的正文和哈希。
- 恢复后 OpenAPI 歌词 meta / 正文重新可用。
- 未绑定歌词记录可删除，已绑定歌词记录删除返回冲突。
- 删除记录不会删除磁盘 `.lrc` 源文件。
