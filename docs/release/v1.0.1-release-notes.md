# Xingyu Music Vault v1.0.1 Release Notes

## 版本号

v1.0.1

## 版本定位

v1.0.1 是一次管理后台视觉与交互体验微调版本，重点完成「仲夏星河 / Midsummer Starlight」主题资源的低风险试接，并将列表页滚动、首页信息展示和 OpenAPI 页面高度控制收口。

本版本不重构主题系统，不修改后端业务逻辑，不接入星语音乐盒项目。

## 主要更新

- 将 `themes/midsummer-starlight/` 主题资源放入 `frontend/public/themes/midsummer-starlight/`
- README 顶部新增主题预览图与试接说明
- 前端入口引用主题 favicon 与 `theme.css`
- 管理后台侧边栏、header、footer 和主内容底板引用主题素材
- 首页迁入音乐统计卡片，并使用主题 banner、logo 和空状态图增强视觉占位
- 全部歌曲、歌手、专辑、歌词、封面及详情列表统一为内部滚动与追加加载交互
- 封面列表收起缩略图，统一通过操作列预览封面
- OpenAPI 页面改为左右两栏内部滚动布局，并隐藏请求示例
- footer 文案改为结构化拼接：项目名、后端版本、当前主题、Apache License 2.0
- `/api/open/v1/server/info` 的 `serviceVersion` 更新为 `1.0.1`
- 移除重复背景素材，旧背景命名改由 `manifest.json` alias 记录
- 生产构建排除主题 `archive/` 备份目录

## 主题资源说明

当前主题资源为生成素材工程化试点版，资源目录为：

```text
frontend/public/themes/midsummer-starlight/
```

保留文件包括：

- `theme.json`
- `theme.css`
- `manifest.json`
- `README.md`

这些文件作为后续轻量主题配置或正式主题系统扩展的基础。

重复背景文件已清理，运行时优先使用：

- `background/background-desktop.webp`
- `background/background-desktop.png`
- `background/background-mobile.webp`

`archive/original-previews/` 保留在源码资源包中，但不进入生产构建产物。

## 已知限制

- Logo 来源于展示板裁切，后续建议重新制作或规范化输出
- 空状态图已用于多个页面，但后续可按页面语义重绘
- 背景图不是原生 4K 终稿
- 当前没有复杂主题切换系统，仅保留当前主题配置读取能力
- 本版本仍不提供在线播放、音频流、客户端写入或公网托管能力

## 验证建议

前端构建：

```bash
cd frontend
npm run build
```

本地开发预览：

```bash
cd frontend
npm run dev
```

重点检查页面：

- 首页
- 全部歌曲
- 歌手列表与歌手详情
- 专辑列表与专辑详情
- 歌词
- 封面
- OpenAPI
