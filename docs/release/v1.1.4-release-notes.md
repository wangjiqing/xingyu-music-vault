# v1.1.4 Release Notes — 反向代理、HTTPS 与非标准端口部署说明

**发布日期：** 2026.06.14

v1.1.4 主要沉淀家庭网络公网访问部署经验，不涉及业务逻辑、数据库结构或 OpenAPI 契约变更。本版本记录星语音库已通过家庭网络反向代理完成公网 HTTPS 访问验证，星语音乐盒已验证出门不挂 VPN 也可以访问星语音库、播放歌曲并加载歌词。

## 推荐链路

```text
https://example.com:18443
→ reverse proxy
→ http://192.168.1.100:18081
→ container:8080
```

说明：

- 公网入口使用 HTTPS 非标准端口 `18443`，适合家用宽带 `80` / `443` 不可用的场景
- 反向代理转发到内网宿主机 `192.168.1.100:18081`
- Docker Compose 默认 `APP_PORT=18081`
- 容器内部 Quarkus 端口保持 `8080`
- 不建议公网直接暴露后端端口

## 安全说明

- 公网访问时建议同时启用管理员登录、OpenAPI AK/SK + HMAC 和 HTTPS
- HMAC 只用于请求认证与防重放，不能替代传输加密
- 证书域名必须匹配访问域名，`CNAME` 不会自动继承证书
- 当前 WebSocket 不是必需项，反向代理无需额外配置 WebSocket 转发
- HSTS 建议在 HTTPS 证书、端口映射和客户端访问都稳定后再启用
- 示例不得替换为真实域名、公网 IP、内网 IP 或证书信息后提交

## 本版本不做

- 不做业务逻辑改动
- 不做 OpenAPI 契约改动
- 不做数据库 migration
- 不做自动证书申请、DNS / DDNS 管理或托管平台封装
- 不做 WebSocket 能力改造
