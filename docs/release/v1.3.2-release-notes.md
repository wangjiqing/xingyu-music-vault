# 星语音库 v1.3.2 - 歌词工作台体验优化

## 发布摘要

v1.3.2 聚焦管理端歌词工作台体验：支持手工粘贴歌词创建草稿，新增 Brave Search 候选来源辅助，管理端工作台支持 SWLRC 逐字歌词试听，并优化草稿编辑、质量摘要、质量报告和 Worker 状态摘要布局。

本版本不修改 Worker 主任务协议，不引入真实进度协议、SSE、文件监听或第三方歌词网页抓取。

## 主要更新

- 支持跳过 Worker 草稿提取，直接粘贴歌词文本创建草稿。
- 手工草稿使用 `LYRIC_DRAFT_MANUAL` 任务类型和 `MANUAL_PASTE` 来源语义，不伪装为 Worker 提取结果。
- 手工草稿可编辑、保存、确认可信歌词，并继续创建逐字歌词对齐任务。
- 新增 Brave Search 配置、状态查询、启用 / 暂停、测试连接和搜索代理接口。
- 草稿页支持 Brave 搜索候选来源，展示标题、域名、摘要和来源链接。
- 支持把候选来源的 provider、query、title、url、domain、selectedAt 等元信息关联到草稿。
- 管理端工作台有 SWLRC 时优先逐字高亮；无 SWLRC、读取失败或解析失败时回退 LRC 行级歌词。
- 草稿核心操作上移，编辑区保持稳定高度；质量摘要、质量报告和 Worker 状态摘要补充中文解释。

## Brave Search 安全边界

Brave Search 用于查找候选歌词来源，不用于自动获取歌词全文。

- 不抓取第三方歌词网页正文。
- 不下载、不缓存、不解析第三方歌词网页全文。
- 不把 Brave 搜索摘要自动写入歌词正文。
- 不把搜索结果片段当作可确认歌词。
- 用户需要自行打开外部来源页面、判断可信度并手工复制歌词文本。
- Brave API Key 不会在接口响应、页面回显或错误信息中返回完整值。

## 配置说明

Brave Key 支持两种模式：

```dotenv
# 环境变量模式，优先级最高；不要在示例、日志或 Git 中写真实 Key
MUSIC_VAULT_BRAVE_SEARCH_API_KEY=

# 控制台托管模式使用该 Key 做服务端加密；请替换为部署环境独有的强随机值
MUSIC_VAULT_SETTINGS_ENCRYPTION_KEY=change-me-settings-encryption-key
```

优先级与行为：

- 设置 `MUSIC_VAULT_BRAVE_SEARCH_API_KEY` 时，实际搜索使用环境变量 Key。
- 环境变量模式下，控制台不会显示完整 Key，也不能通过控制台暂停或替换该环境变量 Key。
- 未设置环境变量时，可在控制台保存托管 Brave Key。
- 控制台托管 Key 使用 `MUSIC_VAULT_SETTINGS_ENCRYPTION_KEY` 加密保存。
- 未配置 settings encryption key 时，控制台托管保存会被拒绝，避免 SQLite 明文保存 API Key。

## 兼容性说明

- 保留原有 Worker 草稿提取、编辑、确认可信歌词流程。
- 保留原有逐字对齐任务创建、Worker 共享目录、审核和导入流程。
- 保留现有任务状态轮询，不替换为 SSE。
- 保留现有 OpenAPI 和音乐盒读取链路。
- SWLRC 逐字渲染只影响管理端工作台试听；缺失或解析异常时自动回退 LRC。

## 不包含内容

- Worker `progress.json`。
- 真实 Worker 进度展示。
- 伪进度、估算阶段或模拟百分比。
- SSE 或文件监听。
- 第三方歌词网页自动抓取。
- 逐字时间轴编辑器。
- 批量全库对齐。

## 升级建议

1. 备份 `data`、`lyrics`、`artwork` 和现有数据库。
2. 更新镜像版本到 `v1.3.2`。
3. 如需 Brave Search，选择环境变量模式或控制台托管模式，不要把真实 API Key 写入可提交文件。
4. 若使用控制台托管 Brave Key，先配置 `MUSIC_VAULT_SETTINGS_ENCRYPTION_KEY`。
5. 重启服务后进入管理端「设置」检查 Brave Search 状态。
6. 在歌曲工作台验证手工草稿、来源关联、可信歌词确认和逐字对齐任务创建。
7. 对已有 SWLRC 的歌曲验证管理端逐字歌词试听；无 SWLRC 的歌曲应正常回退 LRC 行级歌词。
