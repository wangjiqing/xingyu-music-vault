# v1.1.2 Release Notes — 管理员账号初始化与登录 / 登出

**发布日期：** 2026.06.08

v1.1.2 实现管理端基础访问控制后端能力。本版本聚焦单管理员初始化、登录 / 登出、Session Cookie 登录态和管理端接口保护，不处理 OpenAPI Token，也不改变音乐盒 OpenAPI 契约。

## 新增能力

- 首次启动无管理员账号时，可初始化第一个管理员账号
- 初始化完成后关闭初始化入口，不开放注册，不允许创建第二个管理员
- 登录成功后创建服务端 Session，并通过 `XINGYU_MUSIC_VAULT_SESSION` HttpOnly Cookie 保持登录态
- 支持登出，登出后服务端 Session 失效并清理 Cookie
- 支持 `GET /api/admin/auth/me` 查询当前登录用户
- 未登录访问受保护管理接口返回 401
- 密码使用 PBKDF2 哈希存储，数据库不保存明文密码，接口响应不返回 `passwordHash`
- `/api/open/v1/server/info` 的 `serviceVersion` 更新为 `1.1.2`

## 新增接口

- `GET /api/admin/auth/setup-status`
- `POST /api/admin/auth/setup`
- `POST /api/admin/auth/login`
- `POST /api/admin/auth/logout`
- `GET /api/admin/auth/me`

## 数据库

新增 Flyway migration：

```text
V12__create_users.sql
```

新增 `users` 表，字段包括 `id`、`username`、`password_hash`、`role`、`enabled`、`created_at`、`updated_at`、`last_login_at`，并为 `username` 建立唯一索引。本版本业务上只允许一个管理员账号。

## 安全边界

- 管理端登录态使用 Session + HttpOnly Cookie
- Cookie 默认 `Path=/`、`SameSite=Lax`、本地部署默认不启用 `Secure`
- 如在 HTTPS 反向代理后部署，可通过配置启用 Secure Cookie
- Session 当前保存在服务端内存中，服务重启后所有登录态会失效，用户需要重新登录
- 当前尚未实现登录失败次数限制、临时账户锁定或 IP 级登录限流；后续版本可补充暴力破解防护
- `SameSite=Lax` 适合管理端 SPA 与服务端同 origin 的本地 / 局域网部署；如经反向代理暴露到不同域名，应重新评估 CSRF 风险
- OpenAPI Token 是后续独立能力，不与管理端 Session 混用

## 本版本不做

- 不做多用户
- 不做开放注册
- 不做找回密码
- 不做邮箱验证
- 不做 OAuth2 / OIDC
- 不做 NAS 第三方登录
- 不做细粒度 RBAC
- 不做 OpenAPI Token 新能力
- 不把 JWT 存到 localStorage
- 不引入复杂权限模型
