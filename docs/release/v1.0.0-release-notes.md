# v1.0.0 Release Notes

## 版本号

`v1.0.0`

## 发布日期

待发布

## 版本定位

v1.0.0 是 Xingyu Music Vault 的首个正式稳定版本。本版本不新增业务功能，重点完成正式发布前的 README、文档入口、镜像部署示例、Release Notes、检查清单与版本历史整理。

v0.x 阶段已完成 Docker 一键部署、GHCR 自动发布、Docker Hub 自动发布、OpenAPI 只读服务、以及星语音乐盒局域网联调基础验证。

## 核心能力

- 本地音乐扫描、入库与重复扫描跳过
- 歌曲、歌手、专辑维度的管理后台浏览
- 本地 LRC 歌词导入、绑定与查询
- 本地封面扫描、导入、绑定与文件访问
- 音乐元数据编辑、批量整理、音频 Tag 差异比较与受控写回
- 文件信息、安全删除、回收站与恢复
- 只读 OpenAPI 服务，面向播放器客户端和外部工具集成
- Docker Compose 自托管部署

## 部署方式

推荐使用镜像拉取部署：

```bash
cp deploy/.env.example deploy/.env
cp deploy/docker-compose.image.example.yml deploy/docker-compose.yml
cd deploy
docker compose pull
docker compose up -d
```

源码构建部署仍可使用：

```bash
cp .env.example .env
cp docker-compose.example.yml docker-compose.yml
docker compose up -d --build
```

生产部署推荐固定精确版本 tag：`v1.0.0`。`latest` 适合快速体验，不建议作为长期生产固定版本。

## 镜像信息

- GHCR：`ghcr.io/wangjiqing/xingyu-music-vault:v1.0.0`
- Docker Hub：`wangjiqing/xingyu-music-vault:v1.0.0`
- 系列 tag：`v1.0`
- 浮动 tag：`latest`
- 平台：`linux/amd64`、`linux/arm64`

v1.0.0 tag 发布后，需要验证两个仓库的镜像与 manifest。不要在发布前声明 v1.0.0 镜像已验证。

## OpenAPI 能力

OpenAPI 前缀为 `/api/open/v1`，当前为只读服务。已覆盖：

- 服务信息：`GET /api/open/v1/server/info`
- 音乐库状态：`GET /api/open/v1/sync/state`
- 增量变更：`GET /api/open/v1/sync/changes`
- 曲目列表与详情
- 歌词 meta 与正文
- 封面 meta 与二进制文件
- 歌手、专辑聚合接口
- 本地曲目匹配接口

当前 OpenAPI 不提供音频流、不支持客户端写入元数据、不支持上传音乐。

## 升级说明

- 从 v0.9.9 升级到 v1.0.0 时，推荐先备份 `data`、`config`、`lyrics`、`artwork` 等挂载目录。
- 更新 `IMAGE_TAG=v1.0.0` 后执行 `docker compose pull && docker compose up -d`。
- 如果需要回滚，可将 `IMAGE_TAG` 改回已验证的旧版本 tag。

## 备份提醒

升级前建议备份：

- SQLite 数据库所在的数据目录
- 歌词目录
- 封面目录
- 配置目录
- 重要的 compose 与 `.env` 本地配置

音乐目录默认只读挂载，仍建议在执行音频 Tag 写回相关操作前保留独立备份。

## 兼容性说明

- 运行环境：Docker / Docker Compose v2
- 镜像平台：`linux/amd64`、`linux/arm64`
- 数据存储：SQLite 文件持久化
- OpenAPI：继续使用 `/api/open/v1` 只读契约

## 已知限制

- 不提供播放器与在线播放能力
- 不提供公网 HTTPS、域名反向代理或托管平台配置
- 不提供客户端写入 OpenAPI
- 不提供在线歌词抓取、在线封面刮削或 AI 元数据补全
- 不提供 SBOM 或 cosign 镜像签名
