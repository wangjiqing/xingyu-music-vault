# v1.2.1 Release Notes — 歌曲工作台 MVP：边听边看，只读校验闭环

v1.2.1 新增管理端「歌曲工作台」MVP，让用户可以在管理后台边播放歌曲，边查看数据库元数据、歌词、封面和 OpenAPI 输出预览。该能力用于降低人工检查成本，并为后续确认状态打基础。

## 新增能力

- 侧边栏新增「歌曲工作台」入口。
- 全部歌曲列表新增「工作台」操作，可定位打开当前歌曲。
- 工作台支持当前列表内上一首 / 下一首。
- HTML5 audio 播放器支持播放 / 暂停、进度展示、当前时间 / 总时长和进度跳转。
- 工作台展示数据库元数据、歌词内容、封面图片和格式化 OpenAPI 输出预览。
- 新增管理端受保护音频接口 `GET /api/admin/music/{id}/audio`，按数据库 music id 读取已入库本地文件，并支持 Range 请求。
- 新增管理端调试预览接口 `GET /api/admin/music/{id}/openapi-preview`。
- 新增聚合接口 `GET /api/admin/music/{id}/workbench`。

## 安全边界

- 所有新增 `/api/admin/music/**` 接口均保持管理端登录保护。
- 音频接口只读取数据库已登记音乐文件，并校验真实路径位于配置音乐目录内。
- 不开放任意文件路径读取，不新增匿名资源浏览入口。
- 不改变 `/api/open/v1/**` AK/SK + HMAC 认证策略。
- 本版本仍以本地 / 局域网可信使用为主；公网部署继续建议 HTTPS 反向代理和强凭证。

## 非目标

本版本不包含确认状态、确认按钮、AI 候选、网络刮削、歌词公网请求、音频 Tag 写回、封面写入音频文件、复杂播放队列、播放历史、播放统计或收藏系统。

## 验证

- 新增 `MusicWorkbenchResourceTest` 覆盖工作台数据、登录保护、音频读取边界、缺文件和 OpenAPI 预览结构。
- 前端执行生产构建验证。
- OpenAPI HMAC 回归测试通过。
