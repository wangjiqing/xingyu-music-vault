# v1.1.3 Release Notes — OpenAPI AK/SK 凭证管理与 HMAC 签名认证

**发布日期：** 2026.06.10

v1.1.3 将 OpenAPI 客户端认证从旧静态 Token 升级为 AK/SK + HMAC-SHA256。管理端继续使用账号密码登录 + Session Cookie，OpenAPI 使用独立凭证、timestamp 与 nonce 防重放，并支持 `OPENAPI_READ` / `OPENAPI_WRITE` scope。

## Breaking Change

- `xingyu.openapi.auth.enabled` 与 `xingyu.openapi.auth.token` 仅保留为废弃兼容配置，不再参与 `/api/open/v1/**` 认证
- 升级后必须配置 `xingyu.openapi.credential.master-key` / `XINGYU_OPENAPI_CREDENTIAL_MASTER_KEY`，并在管理后台创建 AK/SK 凭证
- 未配置 master key 时，OpenAPI 返回 `500 OPENAPI_CONFIG_ERROR`
- 旧 `Authorization: Bearer <token>` 与 `X-Xingyu-Api-Token` 不再可用
- 尚未适配 AK/SK + HMAC 的客户端应继续使用 v1.1.2 联调

## 新增能力

- 新增 OpenAPI AK/SK 凭证模型，支持 Access Key、Secret Key、scope、启用状态、过期时间和最近使用信息
- 新增管理端 OpenAPI 凭证页面，支持创建、列表、启用 / 禁用和删除凭证
- 创建凭证时 Secret Key 只返回一次，列表和详情不返回明文或密文
- Secret Key 使用 `xingyu.openapi.credential.master-key` + AES-GCM 加密保存
- `/api/open/v1/**` 全量改为 AK/SK + HMAC-SHA256 签名认证
- HMAC 请求头包括 `X-Xingyu-Access-Key`、`X-Xingyu-Timestamp`、`X-Xingyu-Nonce`、`X-Xingyu-Signature-Version: v1`、`X-Xingyu-Signature`
- 签名原文固定为 `METHOD`、`PATH_WITH_CANONICAL_QUERY`、`SHA256_HEX_BODY`、`TIMESTAMP`、`NONCE` 五行
- 支持 timestamp 默认 ±5 分钟窗口校验，以及同一 Access Key 下 nonce 防重放
- 新增 OpenAPI 错误码：凭证禁用、凭证过期、scope 不足、配置错误等
- 新增管理端服务信息接口 `GET /api/admin/server/info`，用于后台页面展示版本号，避免管理端误调用 OpenAPI 客户端接口

## 修复与优化

- 废弃旧 OpenAPI 静态 Token 认证，不再接受 `Authorization: Bearer <legacy-token>` 或 `X-Xingyu-Api-Token`
- 修复缺少 OpenAPI master key 时管理端创建凭证返回通用 `internal_server_error` 的问题，改为明确返回 `OPENAPI_CONFIG_ERROR`
- 修复管理端登录后 footer 自动请求 `/api/open/v1/server/info` 导致 401 刷屏的问题
- OpenAPI 页面不再自动发起未签名请求，避免误触发管理端登录态处理
- 修复管理端启用 / 禁用凭证接口空 body 会静默禁用凭证的问题，改为返回 400
- OpenAPI HMAC 签名读取 body 增加大小上限，默认 `1048576` 字节，超限返回 `OPENAPI_PAYLOAD_TOO_LARGE`
- OpenAPI scope JSON 解析失败时记录 warning，便于排查损坏数据
- nonce 持久化仅将唯一约束冲突视为重复 nonce，其他数据库异常不再误报为重复 nonce
- 更新 OpenAPI 页面示例和文案为 AK/SK + HMAC 签名
- 本机 `%dev` 配置补充 OpenAPI master key，便于 IDEA / `quarkus:dev` 本地创建 AK/SK
- `.env.example`、Docker Compose 示例和部署文档补充 master key、HMAC timestamp、nonce TTL 等配置项

## 部署说明

生产环境必须配置高强度随机 master key：

```bash
XINGYU_OPENAPI_CREDENTIAL_MASTER_KEY=<high-entropy-secret>
```

该值用于加密已创建凭证的 Secret Key。务必妥善备份；更换后，旧凭证的 Secret Key 将无法解密。

可选配置：

```bash
XINGYU_OPENAPI_HMAC_TIMESTAMP_WINDOW_SECONDS=300
XINGYU_OPENAPI_HMAC_NONCE_TTL_SECONDS=600
XINGYU_OPENAPI_RATE_LIMIT_ENABLED=false
XINGYU_OPENAPI_RATE_LIMIT_REQUESTS_PER_MINUTE=120
XINGYU_OPENAPI_ACCESS_LOG_ENABLED=true
```

## 兼容性说明

- 管理端认证仍使用管理员账号 + Session Cookie
- OpenAPI 不再接受旧静态 Token
- 旧版星语音乐盒客户端需要适配 AK/SK + HMAC 签名后才能直接访问 v1.1.3 OpenAPI
- 适配前可继续使用 v1.1.2 作为联调版本

## 本版本不做

- 不做 OAuth2 / OIDC
- 不做 Secret Key 重置流程
- 不做 Token 自动轮换
- 不做多租户凭证隔离
- 不做星语音乐盒客户端侧适配
