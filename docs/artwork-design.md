## 用户流程

```
音乐列表 → 点击封面区域 → 封面选择/导入弹窗
                              ├── 选择已有封面 → 立即绑定
                              └── 导入本地图片 → 保存到 Artworks 目录 → 入库 → 立即绑定
```

## 导入方式

### 已有封面选择

从 `GET /api/artworks?page=0&size=20&keyword=xxx` 搜索已有封面，点击后调用 `PUT /api/music/{id}/artwork` 绑定。

### 本地图片导入

调用 `POST /api/music/{musicId}/artwork/import`，上传 `multipart/form-data`，字段 `file`，支持 `jpg/jpeg/png/webp`，最大 10MB。后端自动完成：

1. 校验图片类型和大小
2. 复制到受控 `app.artwork.scan-dir` 目录
3. 按歌曲信息生成文件名
4. 计算 SHA-256 哈希去重
5. 写入 `artworks` 表
6. 绑定到当前歌曲

## 文件命名规则

格式：`歌手 - 歌名.ext`

- 特殊字符（`/\:*?"<>|`）会被清洗为空格或下划线
- 文件名过长时截断（保持扩展名）
- 路径冲突时追加序号（`-1`、`-2`）
- 按 SHA-256 hash 去重，hash 相同时复用已有 `artworks` 记录

## 绑定策略

- 同一首歌可以有多张封面图，但只有一张主封面（`is_primary = true`）
- 已有主封面时调用绑定会替换
- 调用解绑 `DELETE /api/music/{id}/artwork` 移除主封面

## 后续规划

### URL 导入封面

未来可设计：`POST /api/music/{musicId}/artwork/import-url`

实现前必须处理：

- SSRF 防护（禁止内网 IP、localhost、私有域名）
- 文件大小限制（防止大文件攻击）
- Content-Type 校验（防止伪造文件类型）
- 下载超时
- 重定向限制（限制跟随次数和跳转到内网）
- 域名/IP 限制（白名单来源）
- 来源记录

### 在线搜索封面
