# v1.0.3 Release Notes

v1.0.3 聚焦 Winter Moonlight / 冬夜雪境 与 Spring Dawn / 春日晨光主题接入，完成四季主题资源与基础主题切换体验闭环。

## 本次新增

- 新增 `frontend/public/themes/winter-moonlight/` 主题资源目录
- 新增 `frontend/public/themes/spring-dawn/` 主题资源目录
- 新增 Winter Moonlight / 冬夜雪境 主题候选配置，Theme ID 为 `winter-moonlight`
- 新增 Spring Dawn / 春日晨光 主题候选配置，Theme ID 为 `spring-dawn`
- 管理后台 header 主题切换入口覆盖四套主题：
  - Spring Dawn / 春日晨光
  - Midsummer Starlight / 仲夏星河
  - Autumn Vinyl / 秋日唱片
  - Winter Moonlight / 冬夜雪境
- 主题切换时同步更新背景、Logo、空状态图、favicon 和 `theme.css`
- 主题选择继续写入浏览器 `localStorage`，刷新页面后恢复上次选择
- 当前默认主题仍为 `midsummer-starlight`
- `/api/open/v1/server/info` 的 `serviceVersion` 更新为 `1.0.3`

## 资源目录

```text
frontend/public/themes/
  spring-dawn/
  midsummer-starlight/
  autumn-vinyl/
  winter-moonlight/
```

四套主题均保持同层级资源分类：

- `banner/`
- `logo/`
- `favicon/`
- `background/`
- `empty-states/`
- `palette/`
- `theme.json`
- `theme.css`
- `manifest.json`
- `README.md`

## 素材缺失或待确认项

- Spring Dawn 源素材缺少 `empty-states/metadata-pending.png`，当前运行时使用同主题 `empty-home.png` 兜底，避免资源 404。
- Spring Dawn 源素材缺少 `empty-states/syncing.png`，当前前端未直接引用，已在 `manifest.json` 中标记。
- Winter Moonlight 的 `background/background-desktop.png` 约 3.1MB，运行时使用 `.webp` 版本；后续可压缩 PNG 或评估是否继续保留。
- Winter Moonlight 深色主题与现有浅色硬编码组件仍需后续做深度视觉适配。
- 四套主题 Logo 与空状态图仍带有生成素材工程化版本限制，后续进入正式主题系统前建议重新规范化输出。
- 冬季、春季背景不是原生 4K 终稿，旧 4K / 2K / 1080p 命名仅通过 manifest alias 记录。

## 兼容性

- 不修改后端业务逻辑
- 不修改 OpenAPI 契约
- 不修改数据库结构
- 不扩展音乐扫描、元数据、歌词、封面等业务能力
- 不引入服务端主题管理系统
