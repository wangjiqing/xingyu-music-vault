# 星语音库 v1.3.1 - 正式对齐歌词资产目录收口

## 发布摘要

v1.3.1 聚焦 v1.3.0 后的歌词对齐资产落点收口：Worker jobs 仍作为中间目录，管理员审核通过后，正式 LRC / SWLRC 会发布到歌词挂载目录中的受控 `alignment` 子目录，并继续以 `ALIGNMENT` 来源写入数据库和当前歌词绑定。

本版本不修改 Worker 协议，不引入 Docker Socket、HTTP 回调、消息队列或数据库队列。

## 主要更新

- 新增 `MUSIC_VAULT_ALIGNMENT_LYRICS_ROOT` 与 `MUSIC_VAULT_ALIGNMENT_LYRICS_SUBDIR` 配置。
- 新的审核导入目标为 `{alignmentLyricsRoot}/alignment/{songId}/{jobId}/lyrics.lrc` 和 `lyrics.swlrc`。
- `alignmentLyricsRoot` 必须等于解析后的某一个歌词扫描根目录；未配置时服务可启动，但新的审核导入会明确拒绝。
- 导入流程使用 staging 目录成对复制 LRC / SWLRC，校验 raw file hash 后整体发布 finalDir。
- 同一 jobId 重复导入时，finalDir 已存在且 LRC / SWLRC hash 一致则幂等成功；文件缺失或 hash 不一致则拒绝覆盖。
- 普通歌词扫描排除受控 alignment 子目录，且只复用 / 刷新 `LOCAL_FILE` 歌词记录。
- 普通删除同步继续只处理 `LOCAL_FILE`，不会因 `ALIGNMENT` 文件缺失自动删除歌词、解绑当前歌词或降级来源。
- 修复对齐导入绑定的客户端匹配语义，确保当前歌词绑定继续使用 `TITLE_ARTIST`。
- Docker Compose 示例将 `/lyrics` 挂载为读写，并设置 `MUSIC_VAULT_ALIGNMENT_LYRICS_ROOT=/lyrics`。

## Docker 升级说明

推荐使用精确版本镜像：

```dotenv
IMAGE_TAG=v1.3.1
ALIGNMENT_WORKER_IMAGE=wangjiqing/xingyu-lyrics-aligner:0.4.0
```

确认主服务配置：

```dotenv
LYRICS_DIR=./lyrics
MUSIC_VAULT_ALIGNMENT_LYRICS_ROOT=/lyrics
```

Compose 模板会将宿主机 `LYRICS_DIR` 挂载到容器 `/lyrics:rw`。新的正式对齐导入路径为：

```text
/lyrics/alignment/{songId}/{jobId}/lyrics.lrc
/lyrics/alignment/{songId}/{jobId}/lyrics.swlrc
```

Worker 仍只通过 jobs 共享目录协作：

```text
音库:   ALIGNMENT_JOBS_DIR:/alignment-jobs
Worker: ALIGNMENT_JOBS_DIR:/jobs
Worker: MUSIC_DIR:/music:ro
```

Worker 不暴露端口，不挂 Docker Socket。

## 兼容性

- 旧 `alignment-assets-dir` 不会被自动迁移、删除或清理。
- 历史 `ALIGNMENT` 歌词仍按数据库中的 `source_path` / `swlrc_path` 读取。
- OpenAPI 契约不新增公开状态字段；SWLRC 文件缺失时继续保持 `wordLyricsAvailable=false` 语义。
- LRC OpenAPI 仍可从数据库 `lyrics.content` 返回内容。

## 验证建议

1. 备份 `data`、`lyrics`、`artwork` 和现有数据库。
2. 更新 `IMAGE_TAG=v1.3.1`。
3. 确认 `/lyrics` 为读写挂载，且 `MUSIC_VAULT_ALIGNMENT_LYRICS_ROOT=/lyrics`。
4. 使用“可信歌词 → 创建任务 → Worker 对齐 → 审核 → 导入”验证完整链路。
5. 检查 `/lyrics/alignment/{songId}/{jobId}` 下同时存在 `lyrics.lrc` 和 `lyrics.swlrc`。
6. 手工触发歌词扫描，确认不会新增重复 `LOCAL_FILE` 记录。
7. 重启音库容器后，通过 OpenAPI 和星语音乐盒验证 LRC / SWLRC 可读取。
